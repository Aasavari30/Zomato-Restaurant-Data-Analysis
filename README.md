# 🍽️ Zomato Restaurant Data Analysis Dashboard

## 📌 Project Overview
This project focuses on analyzing Zomato restaurant data to help the business make **data-driven decisions for expanding new restaurants**.

The goal is to identify:
- Best countries and cities for expansion 🌍
- Customer preferences 🍜
- Pricing strategies 💰
- Impact of online delivery & table booking 📲

---

## 🎯 Problem Statement
Zomato wants to expand its restaurant network, but expanding without proper analysis can lead to:
- High competition ❌
- Poor customer satisfaction ❌
- Wrong pricing strategy ❌

This project uses data analysis to provide **smart recommendations for successful expansion**.

---

## 📊 Dataset Information
The dataset includes:
- Restaurant Name
- Country & City
- Cuisines
- Average Cost for Two
- Price Range
- Ratings & Votes
- Online Delivery & Table Booking

---

## 🛠️ Tools & Technologies Used
- Microsoft Excel 📈
- Pivot Tables
- Data Cleaning Techniques
- Lookup Functions (VLOOKUP/XLOOKUP)
- Conditional Formatting
- Dashboard Design

---

## 🧹 Data Cleaning Steps
- Handled missing values
- Converted data types
- Standardized categorical values (Yes/No)
- Mapped country codes using lookup table

---

## 📈 Dashboard Features

The dashboard includes:

- 🌍 Count of Restaurants (Country-wise)
- 📅 Restaurants Opened (Year-wise)
- ⭐ Average Ratings (Country-wise)
- 🗳️ Average Votes (Country-wise)
- 🍜 Top 10 Cuisines
- 💰 Average Cost for Two (INR)
- 📦 Online Delivery & Table Booking Analysis
- 💸 Price Range Distribution

---

## 🔍 Key Insights

- Some countries have **low competition but good ratings** → ideal for expansion
- Popular cuisines like **North Indian & Chinese** perform better
- Restaurants with **online delivery** tend to have better engagement
- Most customers prefer **low to mid price range**

---

## 🚀 Recommendations

- Expand in countries with **low restaurant density**
- Focus on **popular cuisines**
- Maintain **affordable pricing strategy**
- Enable **online delivery & table booking**
